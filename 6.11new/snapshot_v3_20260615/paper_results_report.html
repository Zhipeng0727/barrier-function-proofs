§0 论文核心论点

Thesis: 对于 barrier/Lyapunov 证书的形式验证，Lean4 是唯一能覆盖三类结构性需求的后端： (1) 参数化 ∀n 族定理，(2) 超越函数精确证明(无 δ-gap)，(3) 分段/非光滑结构性证明。

Method: 我们构建路由代理(routing agent)，将每个问题发送到最廉价的健全后端； Lean 作为梯子的最后一级，只在其他方法都到不了的时候才启用 — 由此精确圈定 Lean 的必要性边界。

§5 dReal δ-Gap 实证

实验: 对 V = x² + y²，验证 Lyapunov 正定性 V > 0 (x≠0,y≠0)。
dReal 结果: delta-sat with delta = 0.0001
原因: V(0,0) = 0，所以 ¬(V>0) 即 V≤0 在 (0,0) 的 δ-邻域内恒为 δ-satisfiable。
结构性限制: dReal 对任何在平衡点为零的 Lyapunov 函数都不可能证正定性。
Conditionz3Interval BnBdRealLean4
V > 0 (positive-definiteness) proved (poly only) proved (any) structurally impossible proved (posdef recipe)
V̇ ≤ 0 (decrease) proved (poly only) partial coverage proved (transcendental) proved (with bridging lemma)

§7b 参数化 ∀n Lean 定理 (旗舰结果)

ParametricForallN.lean — Lean4 编译通过, 0 sorry, 0 errors

structurally impossible for ALL numerical backends: z3/interval/dReal 只能验证固定维度 n=2,3,... 的具体实例。Lean 可以证明"对所有 n ∈ ℕ, 该 Lyapunov 函数非负"。
TheoremStatementProof Technique
volterra_entry_nonneg z - 1 - log z ≥ 0 (z > 0) linarith [Real.log_le_sub_one_of_pos]
volterra_entry_pos_of_ne_one z - 1 - log z > 0 (z > 0, z ≠ 1) linarith [Real.log_lt_sub_one_of_pos]
volterra_term_nonneg w·x*·(x/x* - 1 - log(x/x*)) ≥ 0 mul_nonneg + div_pos
volterra_lyapunov_nonneg ∀ n, Σᵢ wᵢ·xᵢ*·g(xᵢ/xᵢ*) ≥ 0 Finset.sum_nonneg

覆盖 core200 中 16 个 L3-族条目: Lotka-Volterra (L3-003/004), replicator ESS (L3-005/006), SIR/SEIR (L3-007/008), detailed-balanced CRN (L3-009/010), complex-balanced CRN (L3-011~014)。 V(x) = Σᵢ wᵢ·xᵢ*·(xᵢ/xᵢ* - 1 - log(xᵢ/xᵢ*)) 是 Volterra 族的通用 Lyapunov 函数。

∀n parametric theorem 是 Lean 优势的纯粹体现: 一个 55 行 Lean 文件替代了 16 个独立的数值验证运行, 且证明的可靠性等级从 "δ-sound" 或 "unsound sampling" 提升到 "machine-checked"。

§9 Case Study: L4-014 (tanh 阻尼振荡器)

dx/dt = y, dy/dt = -x - tanh(y), V = 0.5x² + 0.5y²

验证条件

Positive-definiteness: V = 0.5x² + 0.5y² > 0 for (x,y) ≠ (0,0) — 平方和,trivial
Decrease: V̇ = x·y + y·(-x - tanh(y)) = -y·tanh(y) ≤ 0 — 需证 y·tanh(y) ≥ 0

各后端表现

BackendStatusDetail
z3unsupportedtanh ∉ NRA (非多项式实数算术)
interval BnBunknowndecrease 方向:区间包络 [-ε,ε] 含正负,判不出
dReal (Docker)unknownpositivity: delta-sat (δ-gap!); decrease: unknown
Lean4proved SOUNDtanh_sign recipe: div_nonneg + sinh_nonneg_iff

证明路径(pipeline trace)

  1. Pipeline baseline (无配方): L1 修复 4 轮 → L2 换策略 2 轮 → L3 换 barrier 2 轮 → ❌ 失败
  2. auto_evolve 蒸馏阻塞: "need 0 ≤ y * Real.tanh y"
  3. mine_recipe (GLM 4-parallel + tanh 族提示): 2/4 成功编译 → tanh_sign 入库
  4. 配方注入后重跑: L1 首试即过 (1 轮, 1 API 调用)
L4-014 是 Lean 必要性的旗舰案例: z3 不支持 tanh, interval/dReal 都 unknown, 只有 Lean + 编译验证的 tanh_sign 桥接引理能给出机器检查的健全证明。 配方一旦挖出即永久复用 — 所有 tanh 阻尼系统受益。

Design Lessons

Recipe > Hint
多 recipe 注入(M: 7/7 perfect, 854 tok)碾压抽象引理名提示(C: 5/7, 1532 tok)。 token -44%, 调用 -50%, 时间 -54%。已验证证明模板是 concrete anchor，模型复制/适配远优于从零生成。
Budget Matters
在 B=4 固定预算下, CoT(每次尝试烧 2 calls, 只剩 2 次机会)是灾难。 任何增加 per-attempt 开销的架构都在小预算下失效。Critic 同理: 开销 > 收益。
Oracle Gating
每个 LLM 输出都经 Lean 编译器验证(strict_sorry=True)。 这是 oracle-gated learning: 生成自由, 但只有编译通过的才被接受。零假阳性。
