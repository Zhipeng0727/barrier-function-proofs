LLM ⇄ Lean 4 反馈机制

barrier / Lyapunov 证书的形式化闭环 · lean_proof.py · 演示 lean_feedback_demo.py

大模型擅长猜出一个证书和证明骨架，但它对“这一步 tactic 到底能不能过”只有概率性的直觉。 Lean 4 编译器是确定性的真值预言机：要么 exit 0 通过，要么吐出精确的错误类别和当前目标状态。 本机制把编译器的诊断逐字回灌到下一轮 prompt，让模型基于事实而非猜测去修复—— 这就是 “generate → compile → feed diagnostics back → repair” 的闭环。

1 · 闭环结构

① LLM 生成 Lean 代码
输入：任务目标、系统动力学、已验证的证书 h、证书族骨架模板。输出：单个 ```lean 代码块。
↓
② 抽取代码 extract_lean_code()
用正则把 ```lean … ``` 围栏里的源码取出，写到 _check_<tag>.lean。
↓
③ Lean 编译 lake env lean
在已构建好的工程里跑编译，捕获 stdout+stderr 与退出码。返回 (ok, diagnostics)。
↓
⟲ 失败则回到 ①
④ 构造反馈 build_repair_feedback()
ok → 完成，证书已被机器核验。
fail → 把错误类别 + 假设 + 开放目标 ⊢ … 包成修复 prompt，append 进对话历史，回到 ①。

2 · 三种编译结局

返回值含义动作
Trueexit 0，无 error / 无 sorry接受，证书形式化完成
False编译器报 error（含目标状态）诊断回灌，触发修复轮
None找不到 lake（工具链缺失）跳过编译检查，不阻塞合成

3 · 反馈消息的设计（关键）

回灌给模型的不是“编译失败了”这种空话，而是编译器原样输出。最有信息量的是 开放目标状态—— 它告诉模型当前还差什么、有哪些可用假设。

The Lean 4 code FAILED to compile. Below are the exact compiler
diagnostics (error class, hypotheses, and the open goal `⊢ ...`).
Read the goal state carefully, then output the FULL corrected Lean 4 code block.

----- lean diagnostics -----
error: linarith failed to find a contradiction
case h
x y : ℝ
a✝ : x ^ 2 + y ^ 2 < 2 * x * y
⊢ False
----------------------------

模型读到“linarith failed + 目标是非线性不等式”，立刻知道该换非线性 tactic 并补一个平方提示。

4 · 真实运行实录（零 API token）

lean_feedback_demo.py 把“LLM”换成两段预置回复，但编译是真的—— 直接命中真实的 失败 → 修复成功 路径，因此不花一个 token 就能验证整条链路。

第 1 轮 · 失败

example (x y : ℝ) :
    x^2 + y^2 ≥ 2*x*y := by
  linarith
linarith 是线性 tactic，无法处理二次目标 → 编译 error，诊断回灌。

第 2 轮 · 成功

example (x y : ℝ) :
    x^2 + y^2 ≥ 2*x*y := by
  nlinarith [sq_nonneg (x - y)]
读完反馈后换 nlinarith 并补 (x−y)²≥0 → exit 0，证书被核验。

终端输出（节选）

┌─ round 1: LLM proposes ─────────────
│   linarith
└──────────────────────────────────
  ❌ Lean rejected it. Feedback that goes back to the model:
     error: linarith failed to find a contradiction
     a✝ : x ^ 2 + y ^ 2 < 2 * x * y
     ⊢ False

┌─ round 2: LLM proposes ─────────────
│   nlinarith [sq_nonneg (x - y)]
└──────────────────────────────────
  ✅ Lean accepted the proof — compile exit 0, no errors.

5 · 提升首轮通过率的两个工程要点

证书族骨架模板

自由生成会陷入“Mathlib 引理名轮盘赌”。lean_proof.py 按证书族（log / trig / exp / entropy / piecewise / poly）匹配骨架，预置正确的引理名与证明套路，显著提高首次编译通过率。

精确 import，禁用 umbrella

本机环境里全量 import Mathlib 不可靠；prompt 强制只 import 需要的具体模块 （骨架已给候选），失败的 import 直接丢弃。
