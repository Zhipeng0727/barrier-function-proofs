这个文档怎么打开

直接双击 使用说明.html 这个文件,会用浏览器打开(不需要联网、不需要后端)。 这一页只是说明书;真正的工具是下面要启动的那个网站。

我(主机)怎么启动分享 给我自己

目标:把本机的网站变成一个公网地址,发给同学。电脑保持开机联网即在线,关机/断网就下线。

① 打开终端,跑一条命令

# 进入项目目录,带上你的智谱 key 和模型,起服务 + 开隧道
cd /Users/zhipeng/Desktop/paper4/6.11new
GLM_API_KEY=你的智谱key GLM_MODEL=glm-5.1 ./webapp/share.sh

脚本会自动:生成一个访问口令(终端会打印)→ 自动挑一个空闲端口(从 8001 起,避开 8000 上的测试服务器)→ 需要时安装 cloudflared → 起网站 → 开公网隧道。两个服务各占一个端口,互不影响。想固定端口就在命令前加 PORT=8005。

② 从终端里拿两样东西发给同学

发什么在终端哪里看
公网网址形如 https://xxxx.trycloudflare.com 的那一行
访问口令脚本打印的 访问口令 (发给同学): ……
想自己指定口令,而不是随机生成:在命令前加 APP_PASSWORD=你定的口令。

③ 想结束分享

在那个终端窗口按 Ctrl + C,网站和隧道一起关闭,网址立即失效。

护栏(默认值,可在命令前加环境变量调):
  • 同时真正运行 4 个作业(MAX_CONCURRENT=4),其余排队 —— 因为 Lean 编译吃 CPU,4 个最稳;扛得住可调高。
  • 排队上限 6 个(MAX_QUEUE=6)→ 系统里同时最多 ~10 人。
  • 每天总运行上限 60 次(DAILY_RUN_CAP=60)→ 万一口令泄露也烧不爆账户。
⚠️ 安全红线: 隧道分享只暴露网页、不暴露 key,是安全的。但不要把这个 git 仓库 push 到 GitHub 或整包发给别人 —— runtime_state.py 里有写死的 API key,push 出去就泄露。 要发代码请先轮换 key、改成只走环境变量。

同学怎么用 给使用者

不用装任何东西,有浏览器就行。

  1. 打开网址:把主机发你的 https://xxxx.trycloudflare.com 粘到浏览器地址栏。
  2. 输入口令:进页面会弹出口令框,填主机给的口令,点「进入」。(口令会记在本浏览器,下次免输。)
  3. 选一个系统:
    • 选已有系统:从下拉框挑一个内置动力系统(推荐,成功率高)。
    • 自定义系统(受限):按 JSON 模板填你自己的系统(最多 4 个状态变量,轮数上限更低,更费 token)。
  4. 调参数:拖「最大 debate 轮数」滑块(越大越可能成功,但更费 token);是否勾「生成 Lean 4 形式化证明」。
  5. 点「▶ 运行合成」。右侧会:
    • 如果有人在跑,先显示 「排队中 · 前面还有 N 个」,轮到你自动开始;
    • 然后实时滚动流水线日志(Proposer / Refuter / Lean4 各阶段);
    • 结束后给出结果卡片:是否求解、用了几轮、Lean 是否编译通过、以及生成的 Lean 证明代码。
请按需试用: 每次运行都在花主机的 GLM token(= 钱)和电脑算力。别空跑、别刷。
如果点了没反应 / 报「需要访问口令」: 口令填错了,刷新页面重输;或主机的电脑已关机/隧道已停,找主机确认。

常见问题

Q:同学是用我的电脑算力,还是他们自己的?

大模型推理在智谱云端(花我的 key),Lean 编译在我的电脑。同学那端只是个浏览器,几乎不耗他们的资源。

Q:我电脑关机了会怎样?

网址立即失效,同学打不开。要长期在线得放服务器,但那需要在服务器上重新 build mathlib(几小时、占几十 G),比较折腾。短期试用就保持电脑开机联网最省事。

Q:能不能换模型?

能。启动命令里改 GLM_MODEL=…(确认是智谱 API 里的准确型号名)。

Q:同时能几个人用?

默认 4 个一起跑、再排 6 个,合计同时约 10 人;再多的人点运行会被提示「排队已满,稍后再试」。想放宽改 MAX_CONCURRENT / MAX_QUEUE。
