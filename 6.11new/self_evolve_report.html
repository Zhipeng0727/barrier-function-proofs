结论（控制变量下的因果归因）

SOS 提示是最大杠杆：给并行加 SOS = D 2/7 → E 5/7（+3）；给顺序加 SOS = B 4/7 → C 5/7。logistic_inv 是 SOS 的判别障碍——只有三个 SOS 架构解出，无提示的全军覆没。
并行不是免费午餐：等预算下，纯并行无提示 D=2/7 最差，连单发 A=3/7 都不如——广度耗预算盲搜，没有提示或深度修复就亏。但 并行+SOS（E）墙钟最快（117s） 且 solve 与顺序+SOS 持平 → 广度的价值是提速，不是提质。
评论家辩论(F)在 B=4 没带来 solve 增益（仍 5/7，且多花 3 次评论调用）——但它把 logistic 从 r2 提前到 r1。剩下两个最难的（tanh_sign 需 DerivHyp+sinh 分解、log_self_lower 需 inv-rewrite 链）即便有 SOS 也顶住了 budget=4：它们正是需要"模型升级回退/集成补位"那一层（已离线攻克入库）。
框架优化决策：① 默认用 E（并行+SOS）——最快且 token 接近最优；预算紧时退化到 C（顺序+SOS）最省 token。② 评论家辩论不设默认（小预算无增益），仅在大预算/"错误可修但微妙"时启用。③ 两个最难障碍交给模型升级回退层（gpt-5.5，relay 恢复后自动；当前离线集成已补）。这正好印证三层分工：SOS+并行 吃掉大多数，escalation 吃残差。

9 · Lean 的必要性边界（论文主线）

论文立场（修正）：多后端路由器不是贡献，是陪衬——它的作用是实证地圈出其他方法的极限，从而证明 Lean 在某些类问题上不可替代。若只讲"五种方法选最便宜的"，审稿人立刻问：那为什么要 Lean？答案：固定低维实例上 SMT/区间确实够用且更快（我们诚实地展示这一点，它界定了"容易区"）；但真正重要的定理——参数化 ∀n 族、无界域、δ-精确保证——任何数值/SMT 方法都够不着,只有证明助手能做。Lean 是方法,其余是基线。

顺带纠正一个概念混淆：SOS（平方和）本质是多项式技术；前文 _SOS_HINTS 里 tanh/log/exp/trig 那些其实是解析桥接引理提示,不是 SOS,只有 poly 那条是真 SOS。

Lean 独占的三类能力（其他方法结构性做不到）

能力为什么 z3/interval/dReal 做不到实测例子
参数化 ∀n 族它们只能验固定维度的单个实例;无法表达"对所有 n 成立"。manifest 里这些条目只标了 instances:[5,8]——数值工具只能逐个查这些维度L5-001/L3-010/L3-005 → 路由 lean@2,参数化超越时 z3 直接从梯子里消失
无界域interval/dReal 需要有界 box;全局 Lyapunov（∀x∈ℝⁿ）超出其能力全局 V>0 / V̇≤0 仅 Lean 可证
δ-精确（无 δ 间隙）dReal 是 δ-complete = δ-健全,只证"不存在比 δ 更糟的违例"。致命点:Lyapunov 正定性 V>0 在平衡点处 V(x*)=0,使 ¬(V>0) 永远 δ-可满足 → dReal 结构性证不了正定性(后端自己把 PSD 委托给 interval)实测(Docker dReal):证 x²+y²>0 返回 delta-sat δ=1e-4 证不了;Lean 用 posdef 配方精确证出
论文主结果：这些参数化例子是真实可引用的定理——复杂平衡质量作用网络的 Lyapunov 函数(L3-010)、复制者动力学 ESS 的 KL 散度(L3-005)、n 体一致性(L7-003)。它们同时是参数化 + 超越 + 无界三重难点,只有 Lean 能证。路由器把它们全部送到 lean@2,且参数化超越时把 z3 移出梯子——这就是 Lean 必要性的客观证据。

覆盖范围才是关键：z3 只覆盖多项式固定实例,interval/dReal 只覆盖有界固定维超越——它们都是"容易区"的局部最优。只有 Lean 同时覆盖容易区 + 硬前沿(参数化 ∀n / 无界 / δ-精确)。论文的对比表不是"选最便宜的",而是"画出其他方法的天花板,显示 Lean 是唯一覆盖全空间的方法"。
让 Lean 在硬前沿真正可用 = 论文的工程贡献：Lean 的短板是"人工写证明太慢"。自进化配方流水线(前 6 轮:挖掘+SOS提示+并行+升级+配方库)正是解决这点——把 tanh/log/exp 等反复出现的桥接引理一次证明、永久复用(SMT 每个实例重做)。本次还修复真缺口:路由器 Lean 后端原包旧 generate_lean_proof,已改接强化版 formalize_certificate(注入 recipe_store+升级);修复后 L4-014 整条梯子 → chosen=lean, SOUND=True。
下一步(贴合论文)：① 把参数化 ∀n 族做成 Lean 主结果——L3-010(质量作用网络)/L3-005(复制者 ESS) 的 Fin n 量化定理,这是数值方法结构性够不着的;② 用用户的 Docker dReal 实测一个紧证书的 δ 间隙,把"δ-健全 ≠ 真健全"做成具体对比图;③ NN 证书(CROWN)留作"未来即便神经证书也需 Lean 做最终 kernel 检查"的论据。
